3. Classical Test Theory (CTT)

Code

comp_data = pilot[["rm", "ce", "cf", "sod"]].dropna()

def cronbach_alpha(df):
    k = df.shape[1]
    item_vars = df.var(ddof=1)
    total_var = df.sum(axis=1).var(ddof=1)
    return (k / (k - 1)) * (1 - item_vars.sum() / total_var)

alpha = cronbach_alpha(comp_data)

# Bootstrap 95% CI (1000 resamples)
rng = np.random.default_rng(42)
boot_alphas = [cronbach_alpha(comp_data.iloc[rng.integers(0, 27, 27)])
               for _ in range(1000)]
ci = np.percentile(boot_alphas, [2.5, 97.5])

# Item-total correlations (corrected)
for col in ["rm", "ce", "cf", "sod"]:
    rest = comp_data.drop(columns=[col]).sum(axis=1)
    r, p = stats.pearsonr(comp_data[col], rest)

# Alpha-if-deleted
for col in ["rm", "ce", "cf", "sod"]:
    alpha_del = cronbach_alpha(comp_data.drop(columns=[col]))

4. Monte Carlo Simulation

Purpose: Assess stability of key parameters before operational deployment, given the small pilot n=27.

Method: Multivariate normal simulation from pilot empirical means and covariance matrix · 500 replicates per condition · seeds: 2024 (n=250), 2025 (n=500)

Code

emp_means = pilot_mc.mean().values   # 7 variables
emp_cov   = pilot_mc.cov().values    # 7x7 covariance matrix

def run_mc(n, n_reps=500, seed=2024):
    rng = np.random.default_rng(seed)
    alphas, pass_rates, r2s = [], [], []
    for _ in range(n_reps):
        sim = rng.multivariate_normal(emp_means, emp_cov, size=n)
        sim_df = pd.DataFrame(sim, columns=mc_cols)
        sim_df[["rm","ce","cf","sod"]] = sim_df[["rm","ce","cf","sod"]].clip(1,5).round()
        sim_df["grand_total"] = sim_df["grand_total"].clip(0, 100)
        alphas.append(cronbach_alpha(sim_df[["rm","ce","cf","sod"]]))
        pass_rates.append((sim_df["grand_total"] >= 60.0).mean() * 100)
        # ... R² via least squares
    return alphas, pass_rates, r2s

Alpha — regression to the mean:

The simulated α (M ≈ 0.862) is lower than the pilot α (0.917). Pass rate — upward shift expected:

The pilot pass rate (48%) is likely downward biased due to the 3 missing grand total scores and the small convenience sample. Simulations project a true pass rate of ~56% at cut=60% across both larger sample conditions. This has direct practical implications: if approximately 56% of candidates will pass at this standard in a broader population, selection ratios and pipeline planning should be based on this figure rather than the pilot's 48%.

R² — widest uncertainty:

The R² confidence interval is the widest of all parameters and shows the greatest improvement from n=250 to n=500 (34% reduction). This highlights that predictive validity is the parameter most sensitive to sample size and most in need of formal validation before conclusions are reported. At n=500, the 95% CI still spans ~12 percentage points around the mean R², which is too wide for precise validity claims.

The boxplot shows a clear gradient (Postgraduate > Undergraduate > Non-degree) in grand total scores, consistent with the expectation that more highly educated candidates perform better on the technical component (Section 1). 8. Item Response Theory (IRT)

Python note: Full parametric IRT (GRM, 2PL, 3PL) requires the R mirt package or specialised Python libraries not available in this environment. The Python pipeline provides logistic regression-based discrimination proxies for Section 2 and simulated 2PL curves for Section 1. Results are directionally consistent with full IRT — use R mirt for publication-quality parameter estimates.

Code

from sklearn.linear_model import LogisticRegression
from sklearn.preprocessing import StandardScaler

# Latent trait proxy = standardised total competency score
theta = StandardScaler().fit_transform(comp_data.sum(axis=1).values.reshape(-1,1)).ravel()

# Section 2 — cumulative logistic (GRM proxy)
# For each competency, fit P(X >= k) for thresholds k = 2, 3, 4
for comp in ["rm","ce","cf","sod"]:
    for k in [2, 3, 4]:

        clf = LogisticRegression(max_iter=1000).fit(theta.reshape(-1,1), y_bin)
        a = clf.coef_[0][0]          # discrimination

All four competency items show high discrimination parameters (ā > 1.80), meaning they reliably differentiate candidates across the latent competency continuum. The category response curves show predominantly the P(X≥4) curve because nearly all candidates scored 3 or higher — the lower thresholds (P(X≥2), P(X≥3)) are so easy they are exceeded by almost everyone, leaving only the transition to "4" as the informative threshold.

Key IRT finding — SOD threshold location:

SOD has a threshold-to-4 at b=0.77, meaning a candidate needs to be nearly 0.8 standard deviations above the mean on the latent trait before there is a 50% chance of receiving a rating of 4. This is higher than the equivalent threshold for CE (b=0.21) and CF (b=0.36), confirming that SOD is rated more stringently by assessors — or equivalently, that the behavioural anchor for a "4" rating on SOD represents a more demanding standard. Combined with the skewness analysis (SOD skew = +0.892), this points to assessor conservatism on the SOD dimension rather than poor item quality.

Section 1 — binary item characteristics (bottom row):

Items range from easy (Q01: b=−2.27 — almost all candidates answer correctly) to hard (Q15: b=1.57 — only above-average candidates answer correctly). This spread across the difficulty range is desirable — a well-designed test samples broadly from the ability continuum.

Q08 flag (a=0.675): Q08 is the only item with discrimination below the minimum acceptable threshold of 0.80. A poorly discriminating item provides little information about candidate ability — candidates at all ability levels have similar probabilities of answering correctly. In an operational test, Q08 would be flagged for revision or replacement. Note that these are simulated items based on pilot MCQ totals — the real item-level data is needed for definitive IRT conclusions.

IRT vs CTT — convergent findings:

Both CTT (item-total correlations) and IRT (discrimination parameters) point to SOD as the weakest but still acceptable component of Section 2. Both approaches support retention of all four items in their current form, with targeted improvements to SOD assessor calibration.

# Standardised betas (scale X and y before fitting)
def std_betas(X, y):
    sc = StandardScaler()
    X_std = sc.fit_transform(X)
    y_std = (y - y.mean()) / y.std()
    X_b = np.column_stack([np.ones(len(X_std)), X_std])
    b, *_ = np.linalg.lstsq(X_b, y_std, rcond=None)
    return b[1:]

Interpretation

CE is the only significant predictor of performance:

Across all three models, CE (Communication Effectiveness) is the strongest and only statistically significant predictor of performance score. In M2:

  • CE: std-β = 0.747, t=2.342, p=0.032 — significant at α=.05
  • All other competencies: p > 0.69 — not significant

This is a striking finding. It means that after controlling for the other competencies, only CE provides unique predictive variance for line manager performance ratings. The bivariate scatter plots confirm this: the CE slope is steeper and more consistent than for RM, CF, or SOD.

Why RM and CF are non-significant despite large bivariate correlations:

RM and CF are highly intercorrelated with CE (r > 0.76) due to the general competency factor identified in the EFA. In a multiple regression, this multicollinearity means their unique variance (variance not shared with CE) is minimal. The negative std-betas for RM (−0.183) and CF (−0.202) in M2 are a classic symptom of multicollinearity — sign reversal indicates their individual contributions are being absorbed by CE.
